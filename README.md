# 🧠 Universal AI Agent State Machine

Универсальный фреймворк для создания автономных AI-агентов на базе LLM (OpenAI, Anthropic, и др.), построенный на архитектуре **State Machine** (Конечный автомат) с поддержкой **обязательной рефлексии** и **сохранением состояния**.

Отлично подходит для задач, требующих многошаговых рассуждений, надежной обработки ошибок и взаимодействия с пользователем в реальном времени.

## 🚀 Ключевые особенности

В отличие от классических циклов `while` и реализаций типа LangChain ReAct, этот агент предлагает принципиально иной подход к надежности:

1.  **Архитектура State Machine (Машина состояний).** Агент не "зависает" в бесконечном цикле. Он переходит из состояния в состояние (`PLANNING` → `EXECUTING` → `SUMMARIZING`), что позволяет прозрачно управлять процессом.
2.  **Persisted State (Сохраняемое состояние).** Агент умеет "засыпать". Если ему нужно подтверждение пользователя, он сохраняет текущий контекст в БД и выгружается из памяти. При следующем сообщении он "просыпается" и продолжает с того же места. **Идеально для серверных приложений.**
3.  **Mandatory Reflection (Обязательная рефлексия).** После каждого вызова инструмента агент принудительно анализирует результат. Это позволяет ему самостоятельно исправлять ошибки и менять план, не вовлекая пользователя.
4.  **Semantic Loop Guard (Защита от циклов).** Агент детектирует зацикливание не только по названию функции, но и по её аргументам. Повторный вызов с теми же параметрами мгновенно блокируется.
5.  **Smart Bypass.** Если задача простая и не требует инструментов, агент пропускает фазу планирования и отвечает сразу, экономя токены и время.

## ⚡ Быстрый старт

Агент спроектирован по принципу инъекции зависимостей. Вам нужно определить всего две функции: `llm_client` (общение с моделью) и `tools_handler` (выполнение ваших функций).

```python
from agent import UniversalAgent
import openai

# 1. Настройка клиента LLM (пример для OpenAI)
def my_llm_client(messages, with_tools=False):
    response = openai.chat.completions.create(
        model="gpt-4-turbo",
        messages=messages,
        tools=[...] if with_tools else None # Передайте ваши схемы инструментов
    )
    return response.choices[0].message.model_dump()

# 2. Настройка обработчика инструментов
def my_tools_handler(function_name, arguments):
    if function_name == "get_weather":
        return "Солнечно, 25 градусов."
    return "Функция не найдена"

# 3. Инициализация и запуск
agent = UniversalAgent(
    llm_client=my_llm_client,
    tools_handler=my_tools_handler,
    system_prompt="Ты полезный ассистент, умеющий проверять погоду."
)

# Запуск задачи
result = agent.run("Узнай погоду и посоветуй, брать ли зонтик.")
print(result)
```

## 🛠 Как это работает (Механика)

Агент проходит через три обязательные фазы:

### Фаза 1: PLANNING (Планирование)
Агент получает запрос и строит нумерованный список действий.
*   Проверяет флаг `REQUIRES_TOOLS`.
*   Если `NO` — выдает прямой ответ (Smart Bypass).
*   Если `YES` — переходит к выполнению.

### Фаза 2: EXECUTING (Выполнение и Рефлексия)
Основной цикл работы:
1.  **Act:** Вызов инструмента.
2.  **Observe:** Получение результата.
3.  **Reflect:** Модель анализирует: *"Удалось ли выполнить шаг? Есть ли ошибки? Нужно ли менять план?"*.
4.  **Loop Check:** Проверка на повторяющиеся действия.

*Если агент ждет подтверждения пользователя, состояние сохраняется, процесс завершается до следующего сообщения.*

### Фаза 3: SUMMARIZING (Итог)
Агент очищает технический контекст (логи, JSON) и генерирует лаконичный ответ пользователю на человеческом языке.

## 📖 Философия кода

Этот скрипт написан для использования в продакшене (платформа ProTalk), поэтому в нем заложены механизмы защиты:

*   **Hard Limits:** Жесткие лимиты шагов (по умолчанию 30) и ошибок (5), чтобы избежать неконтролируемого расхода бюджета.
*   **Context Priming:** Используются специальные системные промпты для переключения режима мышления модели (стратег vs исполнитель).
